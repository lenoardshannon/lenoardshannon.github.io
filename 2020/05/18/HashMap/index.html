HashMap

HashMap构成：Array + List + Red Black Tree

Red Black Tree :当在10亿数据进行不到30次比较就能查找到目标时，不禁感叹编程之魅力！人类之伟大呀！ —— 学红黑树有感。

Conclusion:
1:HashMap刚创建时，table总是null，为了节省空间，当添加第一个元素时，table的容量调整为16
2: 当元素个数大于阈值(16*0.75)=12时，扩充容量后大小为原来的2倍，目的时减少调整元素的个数
3：Jdk 1.8 当每个链表长度大于8，并且元素个数大于等于64时，会调整红黑树，目的时提高执行效率
4：Jdk 1.8 当链表长度小于6时，调整成链表
5：Jdk 1.8之前链表是头插入，之后是尾插

HashSet 和HashMap 联系： HashSet 实际用的就是Map做的
