注解

Annotation 是JDK5.0时引入的一种注释机制,我们会通过反射机制访问注解
Java标注可以通过反射获取标注:可以被其他程序(比如:编译器)读取 Springbbot自动装配就大量使用

元注解

负责解释其他注解的注解
meta-annotation :

  • @Target:用于描述注解的适用范围
  • @Retention:一般源码级别才使用,表示我们的注解在什么地方有效 runtime >class >sources
  • Documented:表示是否将我们的注解挂在JAvaDoc中
  • Inherited:子类可以继承父类注解

反射–重要

把Java从静态语言 转换成了动态语言
动态语言：运行时改变其结构的语言

主要的动态语言：JavaScript C# Python PHP

静态语言：Java，c,c++
将java的准动态语言在反射机制下，变成了动态语言

反射机制：

允许层序在执行时借助于Reflection API取得任何类的内部信息,
Class c =Class forName(“java.lang.String”)
加载完类之后，在堆内存的方法区就产生了一个class类型的对象(一个类只有一个class对象),这个对象就包含了完整的的类的结构信息。我们就可以通过这个对象看到类的结构，这个对象就像一面镜子，透过这个镜子看到类的结构，所以，我们形象的称之为：反射

正常方式： 引入需要的包类名称->通过new实例化->取得实例化对象
反射方式： 实例化对象->getClass()->得到完整的包类名称

反射的优缺点

Advantage:可以实现动态创建对象和编译，灵活性高
Disadvantage:对性能影响大

反射相关的主要API

  • java.lang.Class
  • java.lang.reflect.Method
  • java.lang.reflect.Fiedl
  • java.lang.reflect.Constructor
